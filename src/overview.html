Jest Card Game

Application de jeu de cartes Jest implémentant le pattern MVC (Model-View-Controller). Le jeu supporte plusieurs modes d'interface (console, GUI, hybride) et différentes variantes de règles.

Le projet est organisé en quatre packages principaux :

  • app : Gestion de l'application, du lancement et de la sauvegarde.
  • controller : Contrôleurs gérant la logique de jeu.
  • model : Modèle de données (cartes, joueurs, parties).
  • view : Interfaces utilisateur (console, GUI, hybride).

Architecture principale

Le développement repose sur plusieurs patrons de conception (design patterns) pour assurer la flexibilité du code :

  • Pattern MVC : Pour la séparation stricte entre les données, la logique et l'affichage.
  • Pattern Strategy : Pour définir les comportements d'Intelligence Artificielle des joueurs virtuels.
  • Pattern Visitor : Utilisé pour le calcul complexe des scores en fonction des types de cartes.
  • Pattern Factory : Pour la création dynamique des différentes vues (Console ou Swing).
